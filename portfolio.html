Meu Portfólio

Estou desenvolvendo um projeto para facilitar o acesso aos estádios do Rio de Janeiro. O projeto se chama Acesse Rio e consiste em um site que terá diversas funções para auxiliar o torcedor. Teremos ingressos digitais por QR-code, o torcedor vai comprar o ingresso de determinado setor e será gerado um QR-code, ao chegar no estádio o usuário vai escanear seu celular com o QR-code na catraca e será liberado. Também teremos informações sobre horários mais confortáveis para ir de carro ao estádio, e se locomover a pé nos arredores do estádio. Teremos mapas de todos os estádios para facilitar a locomoção. Teremos também as melhores linhas de ônibus e metrô para parar na porta do estádio. E diversas outras funções para ajudar o torcedor a ter uma experiência agradável no jogo do seu time.

Tecnologias utilizadas

HTML (HyperText Markup Language), que traduzido para o português significa Linguagem de Marcação de Hipertexto, é uma linguagem de marcação utilizada para estruturar e formatar o conteúdo de páginas da web. Desenvolvida por Tim Berners-Lee no início da década de 1990, o HTML é a base fundamental para a criação e exibição de documentos na World Wide Web. O HTML permite aos desenvolvedores web definirem a estrutura lógica e hierárquica de uma página, organizando o conteúdo de forma semântica e facilitando sua interpretação pelos navegadores. Ele utiliza uma sintaxe simples e consiste em elementos ou tags, que são delimitados por símbolos de abertura e de fechamento (">"). Essas tags podem conter atributos que fornecem informações adicionais sobre os elementos e ajudam a definir seu comportamento ou aparência. A função principal do HTML é definir a estrutura de uma página, dividindo-a em seções lógicas e fornecendo uma hierarquia de elementos. Os elementos básicos incluem cabeçalhos, parágrafos, listas, imagens, links e tabelas, entre outros. Com esses elementos, é possível criar um documento HTML que organiza o conteúdo de forma coesa e compreensível para os usuários. O HTML trabalha em conjunto com outras tecnologias web, como o CSS (Cascading Style Sheets) e o JavaScript, que serão explicados logo após. Enquanto o HTML se concentra na estrutura e organização do conteúdo, o CSS é usado para estilizar e formatar a aparência visual da página, e o JavaScript adiciona interatividade e funcionalidade dinâmica. Através do HTML, os desenvolvedores podem criar páginas da web que sejam acessíveis, otimizadas para mecanismos de busca e compatíveis com diferentes dispositivos e navegadores. A semântica fornecida pelo HTML permite que os motores de busca compreendam melhor o conteúdo da página, ajudando a melhorar a visibilidade nos resultados de pesquisa. Em resumo, o HTML é uma linguagem fundamental para a criação de páginas da web. Ele serve para estruturar e organizar o conteúdo de forma lógica e semântica, permitindo que os desenvolvedores definam a hierarquia dos elementos e forneçam uma base sólida para a estilização e interatividade usando tecnologias complementares. Com o HTML, é possível criar páginas web que sejam eficientes, acessíveis e visualmente atraentes para os usuários da Internet. Embora o HTML seja uma linguagem fundamental para a criação de páginas da web, existem alguns aspectos considerados como desvantagens ou contras em sua utilização. Aqui estão alguns deles:
  1. Limitações de estilização: O HTML, por si só, oferece recursos limitados para estilização visual. Embora seja possível definir atributos como cores de texto e fundo, tamanhos de fonte e espaçamentos básicos, a formatação avançada e complexa pode ser desafiadora de ser alcançada somente com o HTML. Para estilizar de forma mais elaborada e personalizada, é necessário complementar o HTML com o uso de CSS.
  2. Complexidade para layouts complexos: Quando se trata de criar layouts mais complexos, como grids ou posicionamentos avançados de elementos, o HTML pode ser menos intuitivo e mais trabalhoso. Embora seja possível alcançar esses layouts com o uso de elementos HTML e estilos CSS, pode ser necessário um esforço adicional para obter o resultado desejado.
  3. Manutenção em grande escala: Ao lidar com projetos web extensos e com uma grande quantidade de páginas, a manutenção do HTML pode se tornar um desafio. Caso ocorram mudanças na estrutura básica do site, como a alteração de um cabeçalho ou rodapé, é necessário atualizar manualmente todas as páginas do site para refletir essas mudanças. Isso pode ser demorado e propenso a erros.
  4. Acessibilidade limitada: Embora o HTML forneça recursos básicos de acessibilidade, como tags semânticas para descrever a estrutura do conteúdo, ainda pode ser necessário adotar técnicas adicionais para garantir uma experiência acessível para todos os usuários. Algumas funcionalidades avançadas, como a criação de formulários acessíveis, podem exigir mais esforço e conhecimento específico.
  5. Dependência de suporte do navegador: Embora o HTML seja amplamente suportado pelos navegadores modernos, pode haver diferenças sutis de implementação e suporte para recursos específicos. Isso pode resultar em páginas que são renderizadas de maneira inconsistente ou que funcionam de forma não esperada em diferentes navegadores. Testar e garantir a compatibilidade em vários navegadores pode ser um desafio.
  6. Segurança: O HTML não foi projetado com recursos de segurança robustos. É importante tomar precauções adicionais ao lidar com a entrada de dados do usuário, como evitar vulnerabilidades de ataques de injeção de código ou de cross-site scripting (XSS). É necessário utilizar técnicas e práticas recomendadas para garantir a segurança adequada das aplicações web.
Apesar dessas desvantagens, é importante destacar que o HTML continua sendo uma peça fundamental na construção de páginas web e é amplamente adotado devido à sua simplicidade e acessibilidade. Muitos dos desafios mencionados podem ser superados com a combinação adequada de HTML, CSS e JavaScript, além do uso de frameworks e bibliotecas que ajudam a facilitar o desenvolvimento e a manutenção de projetos web. CSS (Cascading Style Sheets), traduzido para o português como Folhas de Estilo em Cascata, é uma linguagem de estilo utilizada em conjunto com o HTML para definir a aparência e o layout de uma página da web. Enquanto o HTML cuida da estrutura e organização do conteúdo, o CSS é responsável por controlar o design, o posicionamento, as cores, as fontes e outros aspectos visuais da página. O CSS foi desenvolvido para separar o conteúdo da apresentação, proporcionando maior flexibilidade e facilidade na estilização das páginas da web. Com o CSS, os desenvolvedores podem criar regras e estilos que são aplicados a elementos específicos no HTML. Essas regras são definidas em um arquivo CSS separado ou podem ser incorporadas diretamente na tag "style" dentro do documento HTML. O principal objetivo do CSS é permitir que os desenvolvedores apliquem estilos consistentes e padronizados em todas as páginas de um site. Isso facilita a manutenção e a atualização do design, uma vez que as alterações podem ser feitas em um único local e aplicadas automaticamente a todas as páginas que utilizam a mesma folha de estilo CSS. Uma das principais vantagens do CSS é a capacidade de criar estilos em cascata. Isso significa que é possível definir estilos básicos para elementos específicos e, em seguida, aplicar estilos adicionais ou substituí-los com regras mais específicas. Essa abordagem em cascata permite uma maior flexibilidade e controle sobre a aparência dos elementos. Além disso, o CSS oferece uma ampla gama de propriedades e valores que permitem personalizar a aparência dos elementos. É possível definir cores, tamanhos de fonte, espaçamentos, margens, bordas, sombras, animações e muito mais. Combinando essas propriedades, é possível criar designs sofisticados e adaptáveis, que se ajustam a diferentes dispositivos e tamanhos de tela. Outra funcionalidade importante do CSS é a capacidade de criar layouts responsivos. Com o uso de técnicas como media queries, é possível adaptar o design da página de acordo com as características do dispositivo em que está sendo visualizada. Isso permite que a página se ajuste automaticamente para oferecer a melhor experiência de usuário em dispositivos móveis, tablets e desktops. O CSS também desempenha um papel crucial na acessibilidade da web. Ele permite que os desenvolvedores criem páginas acessíveis, fornecendo informações adicionais para tecnologias assistivas. Através de seletores e pseudo-classes, é possível aplicar estilos específicos para usuários com deficiências visuais, oferecendo contraste adequado, tamanhos de fonte ajustáveis e outras melhorias de acessibilidade. Em resumo, o CSS é uma linguagem de estilo que trabalha em conjunto com o HTML para definir a aparência e o layout de uma página da web. Ele permite a criação de designs consistentes, personalizados e adaptáveis, além de desempenhar um papel importante na acessibilidade da web. Com o CSS, os desenvolvedores têm controle total sobre a apresentação visual de suas páginas, garantindo uma experiência agradável e atraente para os usuários. Embora o CSS seja uma linguagem poderosa para estilizar e personalizar a aparência das páginas da web, também apresenta alguns contras ou desvantagens. Aqui estão alguns deles:
  1. Curva de aprendizado: O CSS possui uma curva de aprendizado relativamente íngreme, especialmente para iniciantes na área de desenvolvimento web. Compreender totalmente os conceitos, a sintaxe e as propriedades do CSS pode exigir tempo e prática. Além disso, à medida que os projetos web se tornam mais complexos, é necessário aprender técnicas avançadas e acompanhar as atualizações e novidades do CSS.
  2. Especificidade e hierarquia: O CSS segue regras de especificidade e hierarquia para determinar quais estilos serão aplicados a um elemento quando há conflitos. Essas regras podem ser confusas e difíceis de gerenciar em projetos maiores. Caso as regras de especificidade não sejam entendidas corretamente, podem ocorrer problemas de sobrescrita ou aplicação incorreta de estilos, resultando em comportamentos inesperados.
  3. Manutenção e reutilização: À medida que um projeto web cresce em escala e complexidade, a manutenção do CSS pode se tornar desafiadora. Estilos duplicados ou espalhados por várias folhas de estilo podem tornar as atualizações e modificações mais difíceis e propensas a erros. A falta de organização adequada e a ausência de uma metodologia de desenvolvimento podem dificultar a reutilização de estilos e a manutenção consistente do código.
  4. Inconsistências entre navegadores: Embora os navegadores modernos suportem a maioria das especificações do CSS, ainda podem ocorrer inconsistências na renderização dos estilos em diferentes navegadores. Isso pode levar a diferenças visuais e comportamentais entre os navegadores, exigindo testes e ajustes adicionais para garantir a compatibilidade em múltiplas plataformas.
  5. Desempenho: O uso excessivo ou inadequado de CSS pode impactar o desempenho da página. Estilos complexos, múltiplas requisições de arquivos CSS e seletores muito específicos podem causar atrasos na renderização e no carregamento da página. É importante otimizar o CSS, reduzir o tamanho dos arquivos e adotar boas práticas para garantir um desempenho adequado.
  6. Compatibilidade com versões anteriores: Algumas propriedades e recursos mais recentes do CSS podem não ser suportados por navegadores mais antigos. Isso pode limitar a utilização desses recursos e exigir a criação de soluções alternativas ou fallbacks para garantir uma experiência consistente em todos os navegadores.
Embora essas desvantagens existam, o CSS continua sendo a principal linguagem para estilizar páginas da web. Muitos dos desafios podem ser superados com boas práticas de desenvolvimento, organização do código, utilização de metodologias como CSS modularizado e frameworks que facilitam a escrita e a manutenção do CSS. JavaScript é uma linguagem de programação de alto nível, interpretada e voltada para a web. Foi criada por Brendan Eich em 1995 e desde então se tornou uma das linguagens mais populares e amplamente usadas no desenvolvimento de aplicações web interativas e dinâmicas. O JavaScript é executado pelos navegadores web e permite adicionar interatividade, comportamento e funcionalidade às páginas da web. Ao contrário do HTML e do CSS, que são usados para estruturar e estilizar o conteúdo, o JavaScript concentra-se em tornar as páginas mais dinâmicas e responsivas, respondendo a ações do usuário e manipulando elementos da página em tempo real. Uma das principais características do JavaScript é sua capacidade de responder a eventos. Os eventos podem ser ações do usuário, como clicar em um botão, passar o mouse sobre um elemento ou enviar um formulário. Com JavaScript, é possível definir funções que serão executadas quando esses eventos ocorrerem, permitindo interações em tempo real e uma experiência mais envolvente para o usuário. Além disso, o JavaScript oferece suporte a manipulação do DOM (Document Object Model), que é a representação em memória da estrutura de uma página HTML. Com o JavaScript, é possível acessar e modificar elementos do DOM, adicionar ou remover conteúdo, alterar estilos e classes, criar animações e realizar uma infinidade de manipulações dinâmicas. Outra capacidade importante do JavaScript é a realização de requisições assíncronas ao servidor por meio de tecnologias como AJAX (Asynchronous JavaScript and XML). Essas requisições permitem que os aplicativos web façam chamadas ao servidor em segundo plano, sem interromper a interação do usuário. Isso possibilita a atualização de partes específicas da página sem recarregar a página inteira, resultando em uma experiência mais fluida e rápida. Além de sua utilização no frontend, o JavaScript também é amplamente usado no desenvolvimento de aplicações backend por meio de plataformas como o Node.js. Com o Node.js, é possível executar JavaScript no servidor, permitindo a criação de aplicativos web completos, desde a interface do usuário até a lógica de negócios e a interação com bancos de dados. O JavaScript possui uma vasta quantidade de bibliotecas e frameworks que facilitam o desenvolvimento de aplicações complexas. Alguns exemplos populares incluem o React.js, AngularJS e Vue.js para o frontend, e o Express.js e o Nest.js para o backend. Essas ferramentas auxiliam na criação de aplicações mais eficientes, modulares e escaláveis. Em resumo, o JavaScript é uma linguagem de programação que permite adicionar interatividade e comportamento dinâmico a páginas da web. Com ele, é possível responder a eventos do usuário, manipular o DOM, fazer requisições assíncronas e criar aplicações web completas tanto no frontend quanto no backend. O JavaScript desempenha um papel fundamental no desenvolvimento de aplicações web modernas e é essencial para criar experiências interativas e envolventes na web. Embora o JavaScript seja uma linguagem de programação poderosa e versátil, existem alguns desafios e desvantagens associados à sua utilização. Aqui estão alguns contras do JavaScript:
  1. Segurança: O JavaScript pode apresentar riscos de segurança, especialmente quando usado de forma inadequada. Vulnerabilidades como injeção de código e ataques de cross-site scripting (XSS) podem ocorrer se não forem implementadas as devidas medidas de segurança. É importante validar e filtrar a entrada de dados do usuário e adotar boas práticas de segurança para proteger contra essas vulnerabilidades.
  2. Dependência do cliente: O JavaScript é executado no lado do cliente, o que significa que depende da capacidade do navegador do usuário para interpretar e executar o código corretamente. Isso pode resultar em diferenças de suporte e comportamento entre navegadores e versões de navegadores, exigindo testes e ajustes adicionais para garantir a compatibilidade.
  3. Desempenho do lado do cliente: À medida que as aplicações web se tornam mais complexas e exigem uma quantidade maior de código JavaScript, o desempenho pode ser afetado. O carregamento e a execução de scripts extensos podem retardar a renderização e a interatividade da página, especialmente em dispositivos com recursos limitados. É importante otimizar o código JavaScript, minimizando-o e utilizando técnicas como carregamento assíncrono para melhorar o desempenho.
  4. Acessibilidade: Embora seja possível criar aplicações web acessíveis com JavaScript, é necessário ter cuidado extra para garantir que a funcionalidade essencial esteja disponível para usuários que dependem de tecnologias assistivas. A falta de suporte adequado ou implementação incorreta de recursos de acessibilidade pode prejudicar a usabilidade para pessoas com deficiências.
  5. Curva de aprendizado e complexidade: O JavaScript possui uma curva de aprendizado considerável, especialmente para programadores iniciantes. A linguagem oferece uma ampla gama de recursos e conceitos, como manipulação do DOM, assincronismo, gerenciamento de eventos e orientação a objetos. Compreender e dominar esses conceitos pode exigir tempo e esforço.
  6. Depuração e testes: Depurar e testar código JavaScript pode ser desafiador, especialmente em projetos complexos. A detecção e correção de erros podem exigir o uso de ferramentas de desenvolvimento específicas, além de técnicas de depuração e testes adequadas. Também pode ser difícil garantir uma cobertura de teste completa, devido à natureza dinâmica e interativa do JavaScript.
Apesar desses desafios, o JavaScript continua sendo uma linguagem essencial para o desenvolvimento web moderno. Muitas das desvantagens podem ser mitigadas com boas práticas de desenvolvimento, testes abrangentes e utilização de frameworks e bibliotecas que ajudam a simplificar e otimizar o processo de desenvolvimento em JavaScript.
